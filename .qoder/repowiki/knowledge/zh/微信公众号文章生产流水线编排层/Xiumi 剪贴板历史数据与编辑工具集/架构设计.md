模块以「可编辑中间层」组织：
- `docx_to_clipboard.py` 是入口脚本，解析 docx XML（`word/document.xml`）→ 段落分类（title/heading/body/empty）→ 生成 `config.json` + 剪贴板二进制 + `manifest.json`。
- `clipboard_data/` 目录为持久化产物：每个 Windows Clipboard Format ID 对应一个 `<id>_<name>.bin` 文件，由 `manifest.json` 索引；`config.json` 供人工修改后通过 `regenerate.py` 重建二进制。
- `export_html.py` / `import_html.py` 构成双向转换层：前者把二进制反序列化为带 `<article id="clipboard-content">` 的可读 HTML，并把原始二进制以 base64 嵌入隐藏 `<script type="application/json" id="cb-raw-data">`；后者从该 HTML 中读取内容区与原始数据，经 `expand_patterns` + `normalize_whitespace` 后再调用 `build_all_formats` 写回剪贴板。
- `restore_clipboard.py` / `save_clipboard.py` 直接读写系统剪贴板（依赖 win32api），作为最终 I/O 边界。
- `_verify_roundtrip.py` 是端到端校验器，对比 roundtrip 前后各格式的 SHA-256，确保 HTML 片段未改动时二进制完全一致。
依赖方向单向：docx → config → binary ↔ HTML export/import → clipboard restore/save，regenerate 仅反向消费 config → binary。
