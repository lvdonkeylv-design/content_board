- 所有剪贴板访问都封装在带重试循环的 `_open_clipboard`/`read_clipboard_html`/`write_clipboard` 函数中，统一捕获异常并通过 `[ERROR]/[WARN]/[INFO]` 前缀打印日志。
- Win32 API 调用一律显式设置 `.restype` 和 `.argtypes`，并使用 `ctypes.c_void_p` 避免 64 位指针截断。
- 全局内存操作遵循 GlobalAlloc → GlobalLock → memmove → GlobalUnlock → SetClipboardData 的标准模式，并在 finally 块中保证 CloseClipboard 释放。
- 资源路径一律通过 `os.path.dirname(os.path.abspath(__file__))` 相对脚本自身定位，避免工作目录依赖。
