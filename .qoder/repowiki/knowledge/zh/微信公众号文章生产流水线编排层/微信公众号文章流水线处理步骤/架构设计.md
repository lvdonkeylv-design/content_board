本模块由 9 个独立可执行的 Python 脚本组成，按 step1_1 → step1_2 → step1_3 → step2_1 → step2_2 → step3 → step4 → step5 → step6 的严格顺序串联，通过 `launch.py` 统一调度。

- 数据契约：所有步骤共享同一份 JSON 元素模型（`elements` 列表，每个元素含 `type`/`index`/`runs` 等字段），step1_1 负责从 docx 构建该模型，后续步骤仅追加/修改字段而不改变结构；step2_2 将 table 元素就地替换为 image 引用，step3 读取 step2 输出生成 HTML。
- 输入/输出约定：每个脚本以同目录 `process/` 子文件夹作为工作区，文件名采用 `stepN_xxx.json/html/png` 前缀命名，便于下游按名称自动发现（如 step6 通过 `find_cover_image` 扫描 `step5_crop_cover.*`）。
- 外部依赖边界：step1_1 依赖 python-docx；step1_2/step1_3/step6 通过 `config` 模块调用大模型 API；step2_2 使用 Selenium + headless Chrome 截图；step4 直接通过 ctypes 调用 Windows user32/kernel32 写入剪贴板；step5 使用 OpenCV/Numpy 做图像裁剪与压缩。
- 容错设计：LLM 调用统一封装 `call_model`（带指数退避重试）；step2_2 无表格时原样复制 JSON 直通下游；step6 对标题字节数、摘要长度做上限截断保护。
