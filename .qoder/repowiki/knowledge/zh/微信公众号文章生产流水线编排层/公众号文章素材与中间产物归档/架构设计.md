本模块是纯数据/产物目录，不含可执行代码，结构由外部 Python 脚本（`step1_*.py`、`step2_table_to_image.py`、`step3_json_to_html.py`、`import_html.py`）驱动，形成固定流水线：
- `content_<YYYYMMDD>_N/`：每篇文章一个工作区，根目录放原始 `.docx` 和封面图；`process/` 下保存各阶段产物。
- `process/step1_1_docx_to_json.json`：docx 解析后的结构化元素列表（paragraph/table/image），每个元素带 `type`、`runs`、`heading_level`、`data` 等字段，作为后续步骤的契约。
- `process/step1_2_split_paragraphs.json` / `step1_3_bold_paragraphs.json`：段落切分与加粗标记两步中间态，便于人工在 JSON 中微调。
- `process/step2_table_to_image.json` + `table/*.html` + `table/*.png`：表格渲染为图片的中间产物，HTML 用于预览，PNG 供最终文章嵌入。
- `process/images/`：正文内嵌图片的本地副本，被 step3 模板以相对路径引用。
- `process/step3_json_to_html.html`：Xiumi 风格的微信公众号富文本模板，包含外层装饰容器与底部尾注，正文区域由脚本从 JSON 注入；`_debug_clipboard_before.html` / `_debug_clipboard_after.html` 记录剪贴板粘贴前后的微信编辑器 HTML 快照，用于比对格式差异。
- `content_20260710_1/process/step4_upload_clipboard.html`：经 `import_html.py` 写回剪贴板的完整富文本片段，可直接粘贴到微信公众号后台。
- `history/文章<MMDD>`：历史成品归档，按日期子目录存放已发布的 docx 与配图。
- `image/`：公共封面/头图资源池，供多篇文章复用。
依赖方向单向：docx → JSON → HTML/PNG，产物之间通过文件名约定（如 `table_1.png`）耦合，无运行时导入关系。
