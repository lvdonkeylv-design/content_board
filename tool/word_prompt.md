【任务名称】公众号 / 资讯类 Word 文章 —— 全网核验 + 结构化重排 + 浅绿主题统一化

【输入】
一份 .docx 文件（含正文、图片，可能含图表 / 表格 / 官方文书截图）。文件路径由我指定。

【一、核验规则】
1. 全网检索文中涉及的机构、人物、金额、日期、任免、排名等关键实体，
   至少交叉验证 50 篇高可信来源，优先级：
     官方（监管批复、公司官网、公司年报、上市股东披露）
     权威财经媒体（界面、财联社、经济观察报、21世纪、北京商报、金融界、
     凤凰财经、观察者网、新浪财经、腾讯财经、澎湃、每经、南都 等）

2. 【严格】只对以下情况追加 (…) 注释；其余一律不加：
   ✔ 数据本身错误（数值、姓名、时间、职位与官方口径不符）
   ✔ 关键事实缺失导致读者产生实质误解
   ✘ 【不加】更精确的日期：如原文"7月" 官方"7月10日"（月已涵盖日）
   ✘ 【不加】等价换算：如"0.16亿元" ≈ "1583.38万元"
   ✘ 【不加】不同统计口径造成的差异：如"平台运营机构17家" vs "合作机构30家（含增信）"
   ✘ 【不加】补充信息：营收变化、增速、履历、背景解读
   ✘ 【不加】对文中数据的重新汇总、罗列或再解释
   ✘ 【不加】仅为让读者"更懂"而添加的评论性文字

3. 若确需追加注释：
   - 追加在该原句末尾，形如 " （据 XX，实际为 XX）"（全角括号）
   - 括号左括号、内部文字、右括号统一 yellow highlight
   - 内容简洁，直接说明"应为 XX"及来源，禁止扩展解读

4. 【铁律】原文一字不改。任何情况下不修改、替换、删除原有字词、标点、换行。
   所有修订仅以"追加"形式呈现。

【二、标题处理】
- 主标题（文章大标题）：文首插入 "# "（加粗普通文本，不套 Heading 样式）
- 小节标题（副标题）：文首插入 "## "
- 二者均为纯文本前缀，不改动原标题文字

【三、图片分流处理】
对每张图片判定类型并按分支处理：

(a) 图片是"表格"（含表头 + 数据网格）
    → 转换为原生 Word 表格（python-docx add_table）
    → 边框用 OxmlElement 手工添加（不依赖 "Table Grid" 样式）
    → 若原图上方带有跨列的表格标题（如"高级管理层"）：
      ★ 必须作为表格第一行的合并单元格（cells[0].merge(cells[1]…)）
      ★ 绝对不允许作为独立段落挂在表格外部或跌落到文末
    → 原图片段落清空 run 后原位替换为该表格

(b) 图片是"图表"（柱/线/饼 / 柱线复合等）
    → 用 matplotlib 重绘为浅绿主题 PNG，通过 image_part._blob 就地替换
    → 【重要】严格按原图上真实出现的数据标签绘制：
        · 原图哪个数据点/柱子上有数字 → 新图对应位置照标
        · 原图未标数字的柱子/点 → 新图一律不标（不得凭视觉估读补数）
    → 不得添加原图未出现的辅助信息（趋势线、平均值、注释框等）

(c) 图片是"文书截图 / 照片 / 公章 / 批复原件"等非表格非图表
    → 原样保留，不做任何改动

【四、浅绿主题（全文档所有表格与图表一致）】
统一配色（十六进制，无 #）：
    表头 / 合并标题深绿底         GREEN_HEADER_FILL = "#6da348" 
    相间行浅绿                   GREEN_STRIPE_FILL = "#E8F5E9"
    边框浅绿                     GREEN_BORDER      = "#A8D5A2"
    图表柱填充                    "#A8D5A2"
    图表柱描边                    "#6FBF73"
    图表折线 / 刻度 / 数值标签 / 图例文字 "#2E7D32"
    背景                         white (figure & axes 均设 white)
    数据点标记                    白心 + 深绿 (#2E7D32) 描边

表格排版细节：
- 表头行：底色 6FBF73，白字加粗，居中
- 合并单元格标题（如存在）：底色 6FBF73，白字加粗 14pt，居中
- 数据行：奇数行留白，偶数行 E8F5E9 相间；文本居中，普通字号
- 边框：所有内外框 A8D5A2 single 4sz

图表排版细节：
- figsize 约 (6.8, 4.2), dpi ≥ 180
- 隐藏 top / right 轴脊；左 / 下轴脊 #B0B0B0
- 图例置于图顶居中，无边框，文字色 #2E7D32
- 双 Y 轴时保留原图刻度范围与主刻度
- 若原图上方标注了净利率 / 员工数等具体读数，仅保留原图真实存在的那部分

【五、输出目录与文件命名】
1. 固定输出根目录（不写到源文件目录）：
       E:\AI\content_board\content_instance\

2. 每次任务在该根目录下新建一个子文件夹作为本次核验产物容器：
       content_{YYYYMMDD}_{N}
   其中：
     - YYYYMMDD：任务执行当天的日期（本地时区，8 位）
     - N：从 1 开始的整数序号，避免同日多次执行冲突
     - 逻辑：if 存在 content_{YYYYMMDD}_1 则尝试 _2、_3 …… 直到找到未被占用的编号

3. 目标文件放在该新子文件夹内，文件名 = 原文件名（不含扩展名）+ "_已核验.docx"
   示例：
     源文件：两年两任首席信息官，三湘银行高管薪酬1184万.docx
     执行时间：2026-07-15，同日首次执行
     实际写入：
       E:\AI\content_board\content_instance\content_20260715_1\
                两年两任首席信息官，三湘银行高管薪酬1184万_已核验.docx

4. 落盘流程：
     - 先确保根目录存在（不存在则 makedirs）
     - 从 N=1 开始 while 循环查找未占用的 content_{YYYYMMDD}_{N} 文件夹并创建
     - 组合目标全路径 → 先写 .tmp → os.replace 到最终路径
     - 若目标文件被 Word 占用（PermissionError）：
         同目录内自动递增文件名后缀，如 _已核验_v2.docx、_已核验_v3.docx ……
         并在结束提示中告知：请关闭 Word、删除历史版本、把 _vN 重命名回原名

5. Python 参考实现片段（供落地时对齐）：
